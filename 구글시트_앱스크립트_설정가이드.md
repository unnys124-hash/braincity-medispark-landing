# 브레인시티 랜딩페이지 구글시트 + Apps Script 설정 가이드

이 문서는 현재 랜딩페이지 [index.html](D:/bie/202607-land-Pyeongtaek/index.html)에서

1. 사용자가 `전화상담` 또는 `관심고객 등록`을 클릭했을 때  
2. 이름 / 연락처 / 관심타입 / 상담목적을 입력하고  
3. Google Sheets에 정보가 저장된 뒤  
4. `010-6689-2348`로 전화 연결되게 만드는 방법

을 단계별로 설명한다.

---

## 1. 준비 파일

현재 폴더에 이미 준비된 파일은 아래와 같다.

- Apps Script 코드: [google_apps_script_Code.gs](D:/bie/202607-land-Pyeongtaek/google_apps_script_Code.gs)
- 랜딩페이지: [index.html](D:/bie/202607-land-Pyeongtaek/index.html)

---

## 2. 전체 동작 구조

흐름은 아래와 같다.

1. 사용자가 랜딩페이지에서 `전화상담` 또는 `상담 신청` 버튼 클릭
2. 이름, 연락처 등 입력
3. 랜딩페이지가 Apps Script Web App URL로 데이터를 전송
4. Apps Script가 Google Sheets의 `braincity_leads` 시트에 한 줄 저장
5. 저장 직후 휴대폰 전화앱이 열리면서 `010-6689-2348`로 연결

즉, `정보 저장 -> 전화 연결` 순서다.

---

## 3. Google Sheets 만들기

### 3-1. 새 스프레드시트 만들기

1. 브라우저에서 [Google Sheets](https://sheets.google.com)로 이동
2. `빈 스프레드시트` 클릭
3. 파일명을 예를 들어 아래처럼 변경

`브레인시티_메디스파크_상담리드`

### 3-2. 시트 이름 바꾸기

하단 시트 탭 이름을 아래처럼 바꾼다.

`braincity_leads`

이 이름은 Apps Script 코드와 맞춰져 있으므로 그대로 쓰는 것이 가장 쉽다.

### 3-3. 스프레드시트 ID 복사하기

브라우저 주소창이 예를 들어 아래처럼 보일 것이다.

```text
https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz1234567890/edit#gid=0
```

여기서 `/d/` 와 `/edit` 사이 문자열이 `SPREADSHEET_ID`다.

예:

`1AbCdEfGhIjKlMnOpQrStUvWxYz1234567890`

이 값을 복사해 둔다.

---

## 4. Apps Script 만들기

### 4-1. Apps Script 열기

1. 방금 만든 구글시트 상단 메뉴에서 `확장 프로그램`
2. `Apps Script` 클릭

새 탭이 열리면 기본 `Code.gs` 파일이 보인다.

### 4-2. 코드 붙여넣기

1. 현재 열려 있는 기본 코드 전체 삭제
2. 로컬 파일 [google_apps_script_Code.gs](D:/bie/202607-land-Pyeongtaek/google_apps_script_Code.gs) 내용을 전체 복사
3. Apps Script 편집기에 붙여넣기

### 4-3. SPREADSHEET_ID 수정

코드 맨 위에 아래 줄이 있다.

```javascript
var SPREADSHEET_ID = "YOUR_SPREADSHEET_ID";
```

이 값을 아까 복사한 실제 스프레드시트 ID로 바꾼다.

예:

```javascript
var SPREADSHEET_ID = "1AbCdEfGhIjKlMnOpQrStUvWxYz1234567890";
```

### 4-4. 저장

상단 저장 버튼 클릭  
프로젝트명은 예를 들어 아래처럼 지정

`braincity-landing-leads`

---

## 5. Apps Script 배포하기

### 5-1. 새 배포

1. 우측 상단 `배포` 클릭
2. `새 배포` 클릭

### 5-2. 유형 선택

1. 톱니 또는 유형 선택 메뉴 클릭
2. `웹 앱` 선택

### 5-3. 배포 옵션 설정

아래처럼 설정한다.

- 설명: `braincity landing lead api`
- 실행 사용자: `나`
- 액세스 권한: `모든 사용자`

### 5-4. 배포 실행

1. `배포` 버튼 클릭
2. 처음이면 구글 권한 승인 창이 뜬다
3. 본인 계정 선택
4. 고급 -> 프로젝트로 이동 -> 허용

배포가 끝나면 `웹 앱 URL`이 나온다.

예:

```text
https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxxxxxxxxxx/exec
```

이 URL을 복사한다.

## 6. 랜딩페이지에 Apps Script URL 연결하기

현재 랜딩페이지 [index.html](D:/bie/202607-land-Pyeongtaek/index.html) 안에는 아래 코드가 있다.

```javascript
const APPS_SCRIPT_URL = "YOUR_DEPLOYED_APPS_SCRIPT_URL_HERE";
```

이 부분을 방금 복사한 실제 Web App URL로 바꾼다.

예:

```javascript
const APPS_SCRIPT_URL = "https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxxxxxxxxxx/exec";
```

중요:

- 따옴표 포함해서 바꿔야 한다.
- URL 끝은 보통 `/exec`로 끝난다.

## 7. 저장되는 컬럼 설명

이 랜딩은 아래 순서로 구글시트에 저장되도록 설계되어 있다.

1. 접수일시
2. 이름
3. 연락처
4. 개발자정보
5. 랜딩페이지명
6. 상담버튼위치
7. 관심타입
8. 상담목적
9. 유입URL
10. Referrer
11. Device
12. UTM Source
13. UTM Medium
14. UTM Campaign
15. UTM Content
16. 전화연결여부

예를 들어 어떤 버튼에서 들어왔는지도 `상담버튼위치`에 저장된다.

예:

- `hero_primary`
- `hero_phone`
- `fixed_phone`
- `bottom_form`

이 정보가 있으면 어떤 CTA가 가장 잘 전환되는지 볼 수 있다.

## 8. 실제 테스트 방법

### 8-1. Apps Script 단독 테스트

배포 후 웹 앱 URL을 브라우저에 열어본다.  
정상이라면 아래와 비슷한 JSON이 보인다.

```json
{"ok":true,"message":"braincity lead api is running"}
```

### 8-2. 랜딩 테스트

1. [index.html](D:/bie/202607-land-Pyeongtaek/index.html)을 브라우저에서 연다
2. `전화상담` 버튼 클릭
3. 이름과 연락처 입력
4. `저장 후 전화 연결하기` 또는 `간단 등록하고 상담받기` 클릭

정상이라면:

- 구글시트에 한 줄이 추가되고
- 휴대폰 또는 전화앱이 `010-6689-2348`로 연결된다

### 8-3. 시트 확인

Google Sheets의 `braincity_leads` 시트를 열면 새 행이 추가되어 있어야 한다.

## 9. 자주 막히는 문제

### 문제 1. 저장이 안 된다

확인할 것:

- `SPREADSHEET_ID`를 실제 값으로 바꿨는지
- 배포를 `웹 앱`으로 했는지
- 액세스 권한이 `모든 사용자`인지
- 랜딩페이지의 `APPS_SCRIPT_URL`을 실제 URL로 바꿨는지

### 문제 2. 권한 오류가 난다

확인할 것:

- Apps Script 최초 배포 때 권한 허용을 끝냈는지
- 배포 후 코드를 수정했다면 `새 버전으로 재배포`했는지

### 문제 3. 구글시트는 저장되는데 전화 연결이 안 된다

확인할 것:

- 데스크톱 브라우저에서는 전화앱이 설치되어 있는지
- 모바일에서 테스트 중인지
- 브라우저가 `tel:` 링크 호출을 막지 않는지

참고:

- 모바일 테스트가 가장 정확하다.

### 문제 4. 버튼 눌렀는데 바로 전화만 되고 저장이 안 된다

현재 수정된 랜딩은 `전화상담 버튼 -> 입력폼 -> 저장 -> 전화연결` 구조다.  
만약 예전 HTML 파일을 열고 있으면 direct `tel:` 링크 버전일 수 있으니 [index.html](D:/bie/202607-land-Pyeongtaek/index.html) 최신 파일을 다시 열어야 한다.

## 10. 실제 운영 시 권장 사항

### 10-1. 테스트용 시트와 운영용 시트 분리

처음에는 테스트용 시트를 하나 더 만들어서 충분히 검증한 뒤 운영용으로 바꾸는 것이 안전하다.

### 10-2. 배포 후 URL 재확인

Apps Script를 새 버전으로 다시 배포하면 URL 또는 버전이 달라질 수 있으므로, 배포 후에는 항상 [index.html](D:/bie/202607-land-Pyeongtaek/index.html)의 `APPS_SCRIPT_URL`을 다시 확인한다.

### 10-3. UTM 붙여서 광고 운영

광고 링크에 아래처럼 UTM을 붙이면 시트에서 매체별 성과를 볼 수 있다.

예:

```text
https://yourdomain.com/?utm_source=naver&utm_medium=cpc&utm_campaign=braincity_open&utm_content=hero_a
```

## 12. 참고

현재 랜딩은 `직접 전화 연결`보다 `고객정보 저장 후 전화 연결`을 우선하도록 이미 수정되어 있다.  
즉, 사용자가 `전화상담` 버튼을 눌러도 바로 전화가 걸리지 않고, 먼저 입력폼이 뜬 뒤 저장 후 연결되게 동작한다.
